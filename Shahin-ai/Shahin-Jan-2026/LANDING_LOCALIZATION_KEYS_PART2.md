## Implementation Notes

### File Organization
All 25 files should follow this conversion pattern:
1. Replace hardcoded text with `@Localizer["Landing_PageName_Section_Element"]`
2. Keep existing CSS/JavaScript as-is
3. Maintain the same HTML structure
4. Use the same pattern as Part 1

### Example Conversion Pattern
**Before**:
```html
<h1>أفضل برنامج GRC</h1>
<p>مقارنة شاملة</p>
```

**After**:
```csharp
<h1>@Localizer["Landing_BestGrcSoftware_Hero_Title"]</h1>
<p>@Localizer["Landing_BestGrcSoftware_Hero_Subtitle"]</p>
```

### Next Steps
1. **DO NOT** add these keys to .resx files yet (as per instructions)
2. Convert each .cshtml file to use these localization keys
3. Test each page after conversion
4. Create a separate task to add keys to .resx files when ready

### Special Cases

#### Dogan Consult Pages (standalone)
These 5 pages use `Layout = null` and are completely standalone HTML documents. Consider:
- Creating a separate localization domain for Dogan Consult
- OR keeping them as-is since they're partner content
- OR converting them but with a `DoganConsult_` prefix instead of `Landing_`

#### Pages with Dynamic Content
Some pages (Blog, CaseStudies, Webinars) will likely have dynamic content from a database. The localization keys provided are for static UI elements only.
