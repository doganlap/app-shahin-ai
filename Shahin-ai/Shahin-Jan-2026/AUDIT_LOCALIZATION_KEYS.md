## Implementation Notes

### Pattern Consistency
All keys follow the pattern: `Audit_Context_Detail`
- **Audit_**: Prefix for audit-specific keys
- **Context**: The functional area (e.g., Status, Type, Finding)
- **Detail**: The specific item (e.g., Planned, Internal, Critical)

### Parameterized Keys
Some keys accept parameters for dynamic content:
- `Audit_DeleteWarning` - Takes name and number
- `Audit_DeleteFindingsWarning` - Takes findings count
- `Audit_NoTypeFound` - Takes audit type
- `Audit_ForAuditId` - Takes audit ID
- `Audit_InDays` - Takes number of days

### Shared Keys
Common keys used across multiple views are prefixed without "Audit_":
- `Actions`, `Edit`, `Details`, `Delete`, `Cancel`
- `BackToList`, `SaveChanges`, `Description`, `Statistics`
